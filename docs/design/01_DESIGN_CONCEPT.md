# Design Concept

## 컨셉명

- English: `Fair Bazaar Companion`
- Korean: `공정한 바자르 동행자`

BozorCheck AI는 시장 안에서 사용자를 대신해 상인을 평가하는 앱이 아니다. 사용자가 현재 시세의 기준선을 빠르게 확인하고, 상인과 더 차분하게 대화할 수 있도록 돕는 동행자다.

## 핵심 키워드

- `Trust`: 가격 판단의 근거를 숫자, 신뢰도, 출처로 투명하게 보여준다.
- `Local`: 우즈베키스탄 바자르, 온라인몰, 현지 언어, 흥정 문화를 존중한다.
- `Simple`: 현장에서 빠르게 읽고 입력할 수 있는 단순한 화면을 만든다.
- `Friendly`: 공격적 표현 대신 안내형 문구를 사용한다.
- `Data-driven`: 감정적 판단이 아니라 `fairLow`, `fairMid`, `fairHigh`와 데이터 출처를 기준으로 설명한다.

## 서비스 톤앤매너

- 짧고 직접적이다.
- 사용자를 겁주지 않는다.
- 상인을 비난하지 않는다.
- “사라/사지 마라”처럼 명령하지 않는다.
- 가격 범위와 비교 행동을 제안한다.
- 데이터 부족 시 확신하는 말투를 피한다.

## 사용자가 느껴야 할 감정

- “지금 가격이 대략 어느 정도인지 알겠다.”
- “상인과 싸우지 않고 차분하게 물어볼 수 있겠다.”
- “이 판단은 앱의 감이 아니라 데이터 근거가 있구나.”
- “내 제보가 검토를 거쳐 더 나은 시세에 도움이 되겠구나.”

## 피해야 할 인상

- 상인 감시 앱
- 신고/처벌 앱
- 관광객 전용 바가지 탐지 앱
- 실제 가격을 단정하는 앱
- AI가 모든 답을 알고 있다고 주장하는 앱
- 복잡한 금융/통계 대시보드처럼 보이는 앱

## 심사위원에게 어필할 디자인 포인트

- 사회적 맥락을 고려한 중립적 UX: 상인과 소비자를 대립시키지 않는다.
- 데이터 투명성: 가격만 보여주지 않고 `confidenceScore`, `sourceBreakdown`, `sampleCount`를 함께 보여준다.
- 확장 가능성: mock API에서 실제 Spring API, Dify AI coach, Telegram Bot으로 단계적 확장이 가능하다.
- 현장성: 큰 숫자, 큰 버튼, 짧은 동선으로 시장 안에서 한 손 사용이 가능하다.
- 다국어 구조: Uzbek/Russian/English/Korean 전환을 전제로 문구와 컴포넌트를 설계한다.

## “상인 감시 앱”처럼 보이지 않게 하는 원칙

- `VERY_EXPENSIVE`도 “상당히 높은 편”으로 표현하고 “사기”, “바가지”를 쓰지 않는다.
- 상인 이름, 점포 랭킹, 신고 카운트, 불량 상인 표시를 만들지 않는다.
- 제보는 즉시 시세 반영이 아니라 `PENDING` 검토 상태로 보여준다.
- 가격 판정 근거를 “이 시장의 최근 데이터 기준”으로 표현한다.
- 결과 화면에 “다른 가게와 비교해 보세요”, “중앙값 근처로 흥정해 보세요”처럼 사용자의 다음 행동을 안내한다.
- 빨간색은 최후의 경고가 아니라 “주의해서 비교” 의미로 제한해 사용한다.
