# Copy Guide

## 서비스 전체 말투

- 중립적이고 차분하게 말한다.
- 짧은 문장을 사용한다.
- 판단 근거가 데이터임을 보여준다.
- 사용자가 다음에 할 수 있는 행동을 제안한다.
- 상인과 소비자를 대립시키지 않는다.
- 확신이 낮을 때는 “참고용”이라고 말한다.

## 피해야 할 표현

다음 표현은 서비스 어디에서도 사용하지 않는다.

- “바가지”
- “사기”
- “속았다”
- “사지 마세요”
- “상인이 잘못했습니다”

추가로 피할 표현:

- “이 가격은 틀렸습니다”
- “신고하세요”
- “나쁜 상인”
- “절대 구매 금지”
- “AI가 정확히 판단했습니다”

## 권장 표현

- “좋은 가격입니다.”
- “현재 시세 범위 안에 있습니다.”
- “조금 높은 편입니다.”
- “상당히 높은 편입니다.”
- “중앙값 근처로 흥정해 보세요.”
- “다른 가게와 비교해 보세요.”
- “품질만 확인해 보세요.”
- “참고용으로 확인해 주세요.”
- “검토 후 시세에 반영됩니다.”

## 가격 판정별 문구

### `CHEAP`

“좋은 가격입니다. 품질만 확인해 보세요.”

사용 맥락:

- 결과 badge: “좋은 가격”
- 보조 문구: “현재 데이터 기준으로 낮은 편입니다.”

### `FAIR`

“현재 시세 범위 안에 있습니다.”

사용 맥락:

- 결과 badge: “보통 범위”
- 보조 문구: “최근 데이터 기준으로 무리 없는 범위입니다.”

### `EXPENSIVE`

“조금 높은 편입니다. 중앙값 근처로 흥정해 보세요.”

사용 맥락:

- 결과 badge: “높은 편”
- 보조 문구: “`fairMid` 근처 가격을 기준으로 대화해 보세요.”

### `VERY_EXPENSIVE`

“상당히 높은 편입니다. 다른 가게와 비교해 보세요.”

사용 맥락:

- 결과 badge: “상당히 높은 편”
- 보조 문구: “현재 데이터 기준보다 많이 높습니다.”

## 제보 완료 문구

“제보가 접수되었습니다. 검토 후 시세에 반영됩니다.”

보조 문구:

- “제보 상태: `PENDING`”
- “검토 전 제보는 가격 요약에 바로 반영되지 않습니다.”

## 데이터 부족 문구

“아직 이 시장의 데이터가 충분하지 않습니다. 참고용으로 확인해 주세요.”

보조 문구:

- “표본 수가 적어 가격 범위가 달라질 수 있습니다.”
- “다른 시장 또는 전국 평균도 함께 확인해 보세요.”

## Mock Data 문구

3단계 mock UI에는 다음 문구를 표시한다.

“development mock data only”

한국어 보조 문구:

“개발용 mock data이며 실제 가격이 아닙니다.”

## 버튼 문구

- Home CTA: “가격 확인하기”, “가격 제보하기”
- Product Detail CTA: “이 가격 괜찮나요?”, “가격 제보하기”
- Price Check CTA: “확인하기”
- Report CTA: “제보 제출하기”
- Settings CTA: “최근 검색어 초기화”, “API Status 보기”

## 다국어 문구 기준

- 긴 문구는 2줄까지 허용한다.
- 버튼은 짧은 동사형을 우선한다.
- 누락된 번역은 영어 fallback을 사용한다.
- 상태값 `CHEAP`, `FAIR`, `EXPENSIVE`, `VERY_EXPENSIVE`, `PENDING`은 내부 값으로 유지하고 UI label만 번역한다.
